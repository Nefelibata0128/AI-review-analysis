## 1. 产品概述

### 1.1 一句话描述
一款面向产品经理的 AI 驱动用户评论分析工具，上传用户评论 `.txt` + 产品文档 `.docx/.md`，6 个专用 Agent 协作分析，SSE 实时反馈进度，自动生成包含 KPI 总览、五维特征、流失预警、优先级看板的深度洞察仪表盘。

### 1.2 要解决的问题
产品经理日常面临三个痛点：
- **信息过载**：数百条用户评论散落在应用商店、客服系统、社交媒体，无法系统化处理
- **主观偏差**：凭直觉判断"用户想要什么"，缺乏数据驱动的结构化分析
- **汇报效率低**：从原始评论到可呈现给决策层的分析报告，需要数小时手工整理

### 1.3 目标用户

| 角色 | 场景 | 频率 |
|:---|:---|:---|
| 产品经理 | 版本迭代前分析用户反馈，确定优先级 | 双周/月 |
| 运营/客服负责人 | 监控用户情绪趋势，预警流失风险 | 周 |
| 创业者/创始人 | 验证 PMF，理解早期用户真实诉求 | 月 |
| 数据分析师 | 快速生成结构化 NLP 分析，作为深度分析的前置步骤 | 按需 |

### 1.4 设计原则
- **零门槛**：无需注册、无需配置 API Key、无需理解 Prompt，上传文件即可
- **工业级美学**：Google Material 风格 — 蓝色主色调 `#0050cb`，Inter + Noto Sans SC 字体，12 列网格系统
- **非瀑布流**：左侧导航 + 右侧网格驾驶舱，分区局部滚动，不陷入无尽翻页
- **证据链完整**：每个分析结论必须附带用户原声引用，杜绝 AI 幻觉数据
- **进度可见**：多 Agent 模式下 5 步分析进度实时可见，每步耗时展示

## 2. 系统架构（v3.0）

```
用户浏览器 (localhost:8080)
    │
    │ POST /api/orch/run   (多 Agent 模式，双文件上传)
    │ GET  /api/orch/events (SSE 进度流)
    │ POST /api/*           (Dify 回退，单工作流快速模式)
    │
    ▼
┌──────────────────────────────────────────────────────────┐
│              proxy-server.py (HTTP 代理 :8080)            │
│  ├─ 静态文件服务 (report-dashboard.html)                   │
│  ├─ CORS 自动注入                                         │
│  ├─ .docx 文件解析 (python-docx)                           │
│  ├─ SSE 流式中继                                          │
│  └─ 路由分发: /api/* → Dify | /api/orch/* → orchestrator  │
└────┬──────────────────────┬──────────────────────────────┘
     │                      │
┌────▼──────────┐  ┌────────▼──────────────────────────────┐
│  Dify API     │  │        orchestrator.py                 │
│  (快速模式)    │  │  6 Agent 串行编排 + SSE 事件发射        │
│  1 次调用      │  │                                       │
│  无进度可见    │  │  Agent1 数据预处理 (t=0.3)              │
└───────────────┘  │  Agent2 主观性过滤 (t=0.5)              │
                   │  Agent3 五维特征抽取 (t=0.5)             │
                   │  Agent4 交叉验证 (t=0.3)                 │
                   │  Agent5 业务分析 (t=0.5)                 │
                   │  Agent6 竞品分析 (t=0.5) [第二场景]      │
                   │        │                                │
                   │        ▼                                │
                   │  DeepSeek API (每个 Agent 独立调用)       │
                   │        │                                │
                   │        ▼                                │
                   │  MCP Server :8081 (8 个分析工具)         │
                   └───────────────────────────────────────┘
```

### 两种分析模式

| 模式 | 技术路径 | LLM 调用 | 进度可见 | 适用场景 |
|:---|:---|:---|:---|:---|
| 深度（多 Agent）| orchestrator.py → 5 Agent 串行 | 5 次 | 5 步实时可见 | 完整分析、面试演示 |
| 快速（单工作流）| Dify Workflow API | 1 次 | 仅完成/失败 | 小文件、快速预览 |

### 两个分析场景

| 场景 | 流水线 | 输入文件 |
|:---|:---|:---|
| 用户反馈分析 | Agent1→2→3→4→5 | 用户评论 .txt + 产品文档 .docx/.md (可选) |
| 竞品分析 | Agent1→6→4→5 | 竞品信息 + 产品文档 .docx/.md (可选) |

## 4. UI 设计规范（v3.0）

### 4.1 色彩体系 (Google Material 风格)

| Token | 值 | 用途 |
|:---|:---|:---|
| `--surface` | `#ffffff` | 卡片/面板背景 |
| `--surface-card` | `#f8f9fa` | 次级卡片 |
| `--surface-soft` | `#f1f3f4` | 页面底色 |
| `--surface-dark` | `#1a1c1e` | 侧边栏背景 |
| `--ink` | `#1a1c1e` | 高强调文字 |
| `--body` | `#444746` | 正文 |
| `--mute` | `#757575` | 辅助文字 |
| `--primary` | `#0050cb` | 主色调 / CTA |
| `--hairline` | `#dadce0` | 卡片 1px 边框 |
| `--accent-green` | `#1e8e3e` | 正向指标 |
| `--accent-red` | `#c5221f` | 告警 / 负向指标 |

### 4.2 字体

- **Inter**（Google Fonts CDN）— 英文/数字正文
- **Noto Sans SC**（Google Fonts CDN）— 中文正文
- 回退：`-apple-system, system-ui, PingFang SC, sans-serif`

### 4.3 网格系统

- 12 列 CSS Grid，间距 `16px`
- KPI 卡片：各 3 列（4 个并排）
- 主图表行：8 列 + 4 列（7:3 比例）
- 辅图表行：各 4 列（1:1:1 均等）
- 表格/结论：12 列全宽

### 4.4 布局原则

1. **信息分层**：KPI 顶层 → 图表中层 → 明细底层
2. **F/Z 视觉动线**：最关键 KPI 放左上角
3. **主次面积比**：主图表:辅图表 ≈ 7:3
4. **统一间距**：Zone 间距 `24px`，卡片内边距 `24px`

## 7. 术语表

| 术语 | 说明 |
|:---|:---|
| ABSA | Aspect-Based Sentiment Analysis，目标级情感分析 |
| Agent | 独立的分析单元 = System Prompt + DeepSeek API 调用 + MCP 工具 |
| Orchestrator | 编排引擎，按依赖顺序串行调用 Agent，合并输出 |
| MCP | Model Context Protocol，AI 工具调用的标准协议 |
| SSE | Server-Sent Events，单向流式传输，用于推送分析进度 |
| 五维特征 | 态度 / 目标 / 情绪 / 行为意图 / 信息源 |
| 主观性过滤 | 区分"情绪宣泄"（×0.3 降权）和"理性反馈"（×1.0）的评论分流机制 |
| 评分-文本冲突 | 高分负评 / 低分正评的异常模式检测 |
